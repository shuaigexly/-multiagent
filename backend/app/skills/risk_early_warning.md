# 风险预警矩阵

## 风险评级（概率 × 影响）

|          | 低影响 | 中影响 | 高影响 |
|----------|--------|--------|--------|
| **高概率** | 🟡 中  | 🔴 高  | 🔴 极高 |
| **中概率** | 🟢 低  | 🟡 中  | 🔴 高  |
| **低概率** | 🟢 低  | 🟢 低  | 🟡 中  |

| 评级 | 处理方式 | 响应时间 |
|------|---------|---------|
| 🔴 极高 | 立即上报 CEO，启动应急预案 | 24 小时内 |
| 🔴 高  | 专人负责，每周汇报进展 | 本周内 |
| 🟡 中  | 纳入月度追踪，有缓解计划 | 本月内 |
| 🟢 低  | 记录存档，季度复盘 | 季度内 |

## 经营预警先行指标

### 增长预警
- 连续 2 周新用户环比下降 > 10% → 获客渠道问题
- 活跃用户/注册用户比低于历史均值 -20% → 激活漏斗问题
- NPS 连续 3 个月下降 → 产品满意度危机

### 财务预警
- Burn Rate 超预算 20%+ 且连续 2 个月 → 成本失控
- 应收账款周转天数突增 > 50% → 客户回款风险
- 大客户（占收入 > 15%）出现续约疑虑信号 → 收入集中度风险

### 团队预警
- 核心岗位季度离职率 > 20% → 组织稳定性风险
- 关键项目 2 个以上里程碑延期 → 交付能力不足
- 招聘完成率 < 60% → 后续交付能力缺口

### 竞争预警
- 竞品完成重大融资（有资本加速能力）
- 竞品推出直接对标我们核心功能的产品
- 我们的核心关键词搜索排名被竞品超越

## 风险缓解策略模板
每个高/极高风险必须包含以下5项：
1. **风险描述**：具体什么情况会发生
2. **触发条件**：什么信号代表风险已经触发
3. **缓解措施**：提前能做什么降低概率或影响
4. **应急预案**：风险触发后 72 小时内的行动清单
5. **负责人**：谁对这个风险跟踪负责

## 风险表述规范
✅ 好："若用户增长维持当前趋势，Q4 将缺口 23%，需在 8 月前追加投放预算或下调目标"

❌ 差："增长存在风险，需要关注"

规则：每个风险陈述必须包含**具体数字 + 时间窗口 + 所需决策/资源**
