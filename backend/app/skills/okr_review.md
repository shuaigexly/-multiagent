# OKR 健康评估技能

## 核心公式：进度比

```
进度比 = 实际完成% ÷ 应完成%
应完成% = 当前周次 ÷ 本季度总周数
```

例：第7周（共13周），KR 实际完成 40%
应完成 = 7/13 = 54%，进度比 = 40/54 = 0.74 → 🟡 需关注

## 评级标准

### Key Result 评级
| 评级 | 进度比 |
|------|--------|
| 🟢 按计划 | ≥ 0.9 |
| 🟡 需关注 | 0.6 – 0.9，有缓解措施 |
| 🔴 落后 | < 0.6，或无明确回补计划 |

### Objective 整体评级
| 评级 | 标准 |
|------|------|
| 🟢 健康 | 80%+ 的 KR 绿灯 |
| 🟡 警示 | 50-79% 绿灯，但有重要 KR 红灯 |
| 🔴 危险 | <50% 绿灯，或有 P0 KR 严重落后 |

## OKR 质量诊断清单
- [ ] 每个 O 有明确的"为什么这个季度这件事最重要"
- [ ] KR 是可量化的数字目标（不是活动描述）
- [ ] KR 在团队控制范围内（不依赖外部不可控因素）
- [ ] 每个 O 下 KR 数量 ≤ 5 个
- [ ] KR 之间无重叠（MECE）

## OKR 对齐检查（季度初）
1. 每个团队 KR 能否追溯到公司级 O？
2. 跨团队 KR 依赖是否明确标注并确认？
3. 资源分配比例是否与 O 优先级匹配？
4. 是否有公司级 O 没有任何 KR 承接？（等于没人负责）
