# 飞书上下文使用规则

## 三类数据提取框架

### 📅 日历数据
提取信号：
- 关键会议节点（评审、发布、复盘）→ 判断业务节奏
- 时间密度异常 → 识别资源瓶颈
- 重复型事件缺失 → 判断流程是否中断

使用方式：将日历事件映射到分析时间轴，解释数据波动的业务背景

### ✅ 任务数据
提取信号：
- 未完成任务数量与截止日期 → 执行风险评估
- 任务所有人分布 → 负载均衡问题
- 逾期任务聚集领域 → 组织瓶颈定位

使用方式：将任务状态作为"执行力"指标，与业务结果相关联

### 📄 云文档数据
提取信号：
- 文档内容 → 历史决策背景、指标口径、业务方案
- 最近修改时间 → 工作活跃度
- 文档质量 → 知识沉淀能力

使用方式：文档是最重要的上下文来源，必须提取其中的具体数字、决策、承诺

## 飞书数据缺失时的处理
若 feishu_context 为空或各类数据均为空列表，必须在报告中明确写出：
"飞书上下文不足，无法验证 [具体内容]，建议补充 [具体数据类型]"
