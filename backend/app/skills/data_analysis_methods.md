# 数据分析方法论

## 1. 指标拆解树（Metric Tree）
任何核心指标 = 乘积分解或加法分解：
- 收入 = 用户数 × 转化率 × 客单价 × 复购率
- 每层拆解后，找出贡献最大/变化最大的子项
- 禁止只说"收入下降"，必须定位到哪个子项驱动

## 2. 归因优先级（Root Cause Hierarchy）
1. 外部因素（市场、竞品、季节）
2. 产品因素（版本变更、功能上线/下线）
3. 运营因素（活动、投放、推送）
4. 数据因素（口径变化、统计异常）

归因时逐层排查，不跳过任何层级

## 3. 异常检测规则
- 环比变化 > 20%：必须解释
- 连续 3 期同向变化：趋势判断
- 与历史同期偏差 > 15%：重点分析
- 多指标同时异常：通常指向系统性问题

## 4. 置信度标注规则
- [确定]：有数据直接支撑
- [推断]：基于间接证据的逻辑推导
- [假设]：缺乏数据，基于经验的合理假设
- [低置信度]：数据质量差或样本量不足
