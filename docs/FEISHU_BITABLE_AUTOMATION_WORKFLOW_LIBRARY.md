# 飞书多维表格自动化与工作流模板库

本文档把这套系统应该配置的自动化和工作流一次性列全，避免后续还靠口头描述。

补充口径：

- 本文档中的 `A* / W*` 已经同步到代码返回的 `native_assets` 蓝图里。
- 当前状态是“蓝图就绪”，不是“飞书云侧对象已自动创建”。
- 验收时请结合 `native_assets.manual_finish_checklist`，在飞书 UI 里逐条补齐。

设计原则：

1. 简单单步动作优先用自动化。
2. 有条件分支、循环、补救逻辑的优先用工作流。
3. 所有流程都尽量回写 `交付动作`、`自动化日志`、`交付结果归档`。

## 1. 自动化模板

### A1 新任务入场提醒

触发：

- `添加记录时`

条件：

- `状态 = 待分析`

动作：

- 发送飞书消息给分析运营群
- 在 `交付动作` 表写一条 `工作流记录`

目的：

- 让新任务进入可见范围

### A2 分析完成自动汇报

触发：

- `新增/修改记录满足条件时`

条件：

- `状态 = 已完成`
- `待发送汇报 = 是`

动作：

- 发送飞书消息或邮件
- 正文引用 `工作流消息包`
- 回写 `交付动作`

### A3 执行任务自动创建

触发：

- `新增/修改记录满足条件时`

条件：

- `待创建执行任务 = 是`

动作：

- 创建飞书任务
- 描述使用 `工作流执行包`
- 设置执行 owner 和截止时间
- 回写 `交付动作`

### A4 复核提醒

触发：

- `到达记录中的时间时`

条件：

- `待安排复核 = 是`

动作：

- 发送提醒
- 创建复核任务
- 回写 `交付动作`

### A5 超时未执行提醒

触发：

- `定时触发`

动作：

- 查找 `执行截止时间 < 当前时间` 且 `是否已执行落地 != 是` 的记录
- 发送提醒给执行 owner

### A6 超时未复核提醒

触发：

- `定时触发`

动作：

- 查找 `建议复核时间 < 当前时间` 且 `待安排复核 = 是` 的记录
- 提醒复核 owner

### A7 失败动作报警

触发：

- `添加记录时`

条件：

- 在 `交付动作` 表里 `动作状态 = 执行失败`

动作：

- 发报警消息
- 进入人工接管群

### A8 归档提醒

触发：

- `新增/修改记录满足条件时`

条件：

- `状态 = 已完成`
- `是否已执行落地 = 是`

动作：

- 通知管理员将结果写入 `交付结果归档`

## 2. 工作流模板

### W1 路由总分发工作流

触发：

- `状态 = 已完成`

逻辑：

- If `工作流路由 = 直接汇报` -> 汇报分支
- If `工作流路由 = 等待拍板` -> 拍板分支
- If `工作流路由 = 直接执行` -> 执行分支
- If `工作流路由 = 补数复核` -> 复核分支
- If `工作流路由 = 重新分析` -> 重跑分支

### W2 汇报分支工作流

动作：

- AI 生成更口语化的管理摘要
- 发送飞书消息
- 发送邮件
- 写交付动作

### W3 拍板分支工作流

动作：

- 用 `一句话结论 + 必须拍板事项 + 汇报风险` 生成高管卡片
- 发送给拍板人
- 等待拍板结果回写

### W4 执行分支工作流

动作：

- 根据 `执行 owner` 查找负责人
- 循环为多个 owner 创建任务
- 若任务创建失败，进入失败补救分支

### W5 复核分支工作流

动作：

- 查找需要补数的条目
- 发送提醒
- 创建复核任务
- 到期再次提醒

### W6 重跑分支工作流

动作：

- 保留旧版本号
- 新建重跑任务
- 在 `复核历史` 写入“重跑原因”

### W7 动作失败补救工作流

触发：

- `交付动作.动作状态 = 执行失败`

动作：

- 条件判断失败类型
- 可重试则重试
- 不可重试则转人工接管

### W8 群消息驱动工作流

触发：

- `接收到飞书消息时`

用途：

- 在群里输入关键词触发复盘、日报、复核、催办

### W9 仪表盘自动推送工作流

触发：

- `定时触发`

动作：

- 推送仪表盘截图
- 附带 AI 智能总结

### W10 数据资产校验工作流

触发：

- `定时触发`

动作：

- 查找长期未校验数据集
- 通知数据 owner

## 3. AI 节点推荐用法

根据官方资料，当前最适合接入本系统的 AI 能力有：

1. `AI 生成文本`
   - 生成高管口径摘要
   - 生成群消息卡片正文
   - 生成邮件主题和正文

2. `AI 分类`
   - 给任务自动打风险标签
   - 给任务自动打业务主题标签

3. `字段捷径`
   - 生成更短的展示摘要
   - 对文本做分类与总结

4. `仪表盘智能解释`
   - 自动解释图表
   - 随仪表盘推送附带洞察

## 4. 推荐命名规范

自动化：

- `AUTO_任务入场提醒`
- `AUTO_分析完成自动汇报`
- `AUTO_执行任务自动创建`
- `AUTO_复核提醒`

工作流：

- `WF_路由总分发`
- `WF_直接汇报`
- `WF_等待拍板`
- `WF_直接执行`
- `WF_补数复核`
- `WF_重新分析`

---

## 5. 最终要求

所有自动化和工作流都应满足三条要求：

1. 能看懂是给谁服务的。
2. 能看懂失败后怎么补救。
3. 能回写到 `交付动作` 或 `自动化日志`，避免“看起来自动化很多，但没人知道它到底跑没跑”。
