# 飞书多维表格验收边界与真实联调清单

本文档用于明确一件事：

仓库内“测试通过”和“真实飞书多维表格全链路跑通”不是同一个结论。

如果后续要对外汇报、交接或验收，必须以本文档为准，不要把本地连通测试误写成真实飞书验收完成。

## 2. 当前没有实际验证的范围

以下内容目前不能宣称“已跑通”，除非拿到真实飞书权限并完成联调：

1. `workflow/setup` 是否真的在飞书云端成功创建 Base、表、字段、视图
2. `workflow/seed` 是否真的写入飞书多维表格记录
3. `workflow/start` 后调度器是否真的持续更新线上记录
4. `工作流消息包 / 工作流执行包` 是否真的被飞书自动化消费
5. 飞书消息、飞书任务、复核提醒等原生动作是否真的执行成功
6. 线上多维表格字段类型、视图配置、权限配置是否与代码假设完全一致

原因不是代码一定有问题，而是：

1. 当前执行环境没有可用的飞书应用调用权限
2. 没有可确认可写的 `app_token`
3. 没有可确认可用的 `tenant_access_token` / `user_access_token`
4. 没有可确认完整的应用 scope

## 3. 对外汇报时的正确口径

当前阶段应该使用下面的口径：

- 已完成：本地代码链路、模板中心逻辑、后端回归测试、前端构建验收
- 未完成：真实飞书多维表格端到端联调验收

不应使用下面的表述：

- “飞书全链路已跑通”
- “真实多维表格写入验收完成”
- “飞书自动化已经验证生效”

---

## 4. 真实飞书联调的最小前提

要开始真实联调，至少需要：

1. 一个可写的飞书自建应用
2. 应用已开通多维表格相关 OpenAPI 权限
3. 应用已完成管理员授权
4. 可用的环境变量：
   - `FEISHU_APP_ID`
   - `FEISHU_APP_SECRET`
5. 如果要验证飞书任务 / 消息 / 用户视角能力，还需要：
   - 可用 `user_access_token`
   - 对应用户 OAuth scope

---

## 5. 建议的真实联调 scope 清单

真实联调至少应核对以下权限是否已申请并生效：

1. 多维表格应用与表结构读写
2. 多维表格记录读写
3. 云空间文件读写
4. 消息发送
5. 飞书任务创建与读取
6. 如果要验证知识库/文档落地，还需要文档相关 scope

具体名称以飞书开放平台控制台实际权限项为准，不要只凭代码猜测。

## 6. 真实飞书端到端验收步骤

### 6.1 基础连通

1. 配置真实飞书应用环境变量
2. 启动后端服务
3. 调 `POST /api/v1/workflow/setup`
4. 确认返回真实 `app_token` 与飞书 Base URL
5. 在飞书客户端打开该 Base，确认 12 张表、字段、视图均已创建

### 6.2 任务写入

1. 调 `POST /api/v1/workflow/seed`
2. 使用显式 `template_name` 写入一条任务
3. 再使用仅 `output_purpose` 写入一条任务
4. 在 `分析任务` 表确认：
   - `套用模板`
   - `汇报对象`
   - `执行负责人`
   - `复核负责人`
   - `复核SLA小时`
   已被正确写入

### 6.3 调度执行

1. 调 `POST /api/v1/workflow/start`
2. 等待至少一条任务从 `待分析` 进入 `分析中`
3. 确认随后产生：
   - `岗位分析`
   - `综合报告`
   - `证据链`
   - `产出评审`
   - `交付动作`
   - `复核历史`
   - `交付结果归档`
   - `自动化日志`

### 6.4 模板驱动交付

1. 在已完成任务主表确认：
   - `工作流路由`
   - `套用模板`
   - `工作流消息包`
   - `工作流执行包`
2. 对照 `模板配置中心` 中对应模板，确认渲染结果符合模板内容

### 6.5 再流转

1. 确认 CEO 行动项生成跟进任务
2. 确认评审建议能生成复核任务
3. 确认这些自动生成任务同样带有模板默认值

### 6.6 飞书原生动作

1. 如果启用了多维表格自动化：
   - 验证消息发送
   - 验证飞书任务创建
   - 验证复核提醒
2. 把执行结果回写到 `交付动作` 或 `自动化日志`

---

## 7. 真实飞书验收通过标准

只有同时满足以下条件，才可以说“真实飞书全链路已跑通”：

1. Base 创建成功
2. 任务 seed 成功写入线上表
3. 调度器成功更新线上表状态
4. 各子表产物真实生成
5. 模板中心真实驱动消息包与执行包
6. 跟进任务 / 复核任务真实生成
7. 至少一条飞书原生自动化或任务动作验证成功

少任何一项，都只能叫“部分联调通过”。
